# Mira 安全政策

## 支持范围

Mira 仍处于快速开发阶段。安全修复优先进入最新发布版本和 `main` 分支；旧测试包不保证继续获得修复。

## 报告安全问题

涉及 API Key、聊天记录、任意文件访问、权限绕过、远程代码执行、插件供应链或备份泄露的问题，不要直接把复现数据贴进公开 Issue。

优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告入口。若该入口暂未开放，请先创建一个不包含漏洞细节、密钥、聊天内容或个人数据的 Issue，请求建立私密沟通渠道。

报告中建议包含：

- 受影响版本、提交或 APK 来源
- Android 版本、设备架构和必要权限状态
- 最小复现步骤与实际结果
- 影响范围和可重复性
- 已脱敏的日志、堆栈或示例文件

## 用户侧安全建议

- 仅从本仓库 Release 或可信的自构建产物安装 APK。
- 为模型和语音服务使用独立、有限额、可撤销的 API Key。
- 不公开模型配置导出、完整备份、原始日志和含通知内容的截图。
- 高权限能力按需开启；无障碍、Shizuku、Root 和终端权限不使用时关闭。
- 安装 MCP、Skill、脚本或工具包前检查源码、作者、更新地址和网络端点。
- 发现异常请求、未知工具调用或费用增长时，先停用扩展并撤销相关 Key。

## 项目维护约束

- 仓库不提交真实 API Key、Token、签名文件、OAuth Secret 或用户数据样本。
- 教程截图只使用脱敏素材；原始设备抓图不作为发布资源。
- 发布 APK 使用固定签名并保留离线备份，版本号只增不减。
- 依赖、插件与上游同步需要保留来源和许可证信息。

## 公开披露

修复完成并发布后再讨论漏洞细节。披露时间根据影响、修复复杂度和用户升级窗口协调确定。
