# Mira 安全政策

## 支持范围

Mira 仍处于快速开发阶段。安全修复优先进入最新发布版本和 `main` 分支；旧测试包不保证继续获得修复。

## 报告安全问题

涉及 API Key、聊天记录、任意文件访问、权限绕过、远程代码执行、插件供应链或备份泄露的问题，不要直接把复现数据贴进公开 Issue。

优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告入口。若该入口暂未开放，请先创建一个不包含漏洞细节、密钥、聊天内容或个人数据的 Issue，请求建立私密沟通渠道。

报告中建议包含：

- 受影响版本、提交或 APK 来源
- Android 版本、设备架构和必要权限状态
- 最小复现步骤与实际结果
- 影响范围和可重复性
- 已脱敏的日志、堆栈或示例文件

## 用户侧安全建议

- 仅从本仓库 Release 或可信的自构建产物安装 APK。
- 为模型和语音服务使用独立、有限额、可撤销的 API Key。
- 不公开模型配置导出、完整备份、原始日志和含通知内容的截图。
- 高权限能力按需开启；无障碍、Shizuku、Root 和终端权限不使用时关闭。
- 安装 MCP、Skill、脚本或工具包前检查源码、作者、更新地址和网络端点。
- 发现异常请求、未知工具调用或费用增长时，先停用扩展并撤销相关 Key。

## 信任边界

- **Mira APK**：只信任本仓库 Release 中使用项目固定签名发布的产物。文件名相同不代表签名相同。
- **模型与语音端点**：Mira 按用户配置直连服务商；中转站、自定义 Endpoint 和兼容 API 属于各自的信任边界。
- **插件与市场**：市场提供发现和元数据，插件代码可能来自作者仓库或第三方 Release。市场展示不等于代码审计结论。
- **GitHub 账号**：Mira 使用 Device Flow，公开 APK 只携带公开 Client ID。Token 只用于 GitHub 官方 API；新市场条目、评论和 Reaction 属于公开 GitHub 内容。
- **高权限通道**：无障碍、Shizuku、Root、终端和工作区可以放大插件或错误操作的影响，授权范围应与任务匹配。
- **备份与导出**：离开 Android 应用私有目录后，不再受应用沙箱保护；当前导出文件不应被当作自动加密文件。

## 发布完整性

正式发布必须使用长期保留的 release keystore，并确保 `versionCode` 单调递增。每个 Release 应只放一个正式 APK，同时公布 SHA-256。签名更换会导致 Android 拒绝覆盖安装，并迫使用户卸载旧版，从而增加本地数据丢失风险。

## 项目维护约束

- 仓库不提交真实 API Key、Token、签名文件、OAuth Secret 或用户数据样本。
- Release 构建不使用 OAuth Client Secret；发布前扫描 APK，确认 GitHub Token、Device Code 和本机配置未进入产物。
- 教程截图只使用脱敏素材；原始设备抓图不作为发布资源。
- 发布 APK 使用固定签名并保留离线备份，版本号只增不减。
- 依赖、插件与上游同步需要保留来源和许可证信息。

## 公开披露

修复完成并发布后再讨论漏洞细节。披露时间根据影响、修复复杂度和用户升级窗口协调确定。
