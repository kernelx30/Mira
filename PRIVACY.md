# Mira 隐私说明

更新日期：2026-07-18

Mira 是本地优先的 Android AI 伴侣。本说明描述当前开源版本的数据流和用户可控边界，不替代第三方模型、语音、搜索或插件服务商自己的隐私政策。

## 默认保存在设备上的数据

- 聊天消息、会话标题、分组和摘要
- 用户资料、角色卡、头像引用和关系状态
- 伴侣记忆、证据、版本和关联
- 模型、TTS、STT、插件和界面配置
- 本地索引、缓存、日志以及用户主动创建的备份

这些数据位于 Android 应用数据目录或用户明确选择的导出位置。卸载应用通常会删除应用私有数据，但不会自动删除已经导出到下载目录或其他存储位置的备份。

## 记忆的隔离与使用

Mira 的伴侣记忆按用途分为四种范围：

- `USER`：用户稳定身份、全局偏好和通用边界，默认可供所有角色使用。
- `COMPANION`：角色自身设定和角色私有认知，默认仅该角色读取。
- `RELATIONSHIP`：用户与某个角色之间的称呼、经历、约定和关系状态，默认仅这一对关系读取。
- `CONVERSATION`：会话摘要、章节和未完成话题，默认仅当前会话参与者读取。

角色私有记忆不会因为切换角色而自动共享。底层支持对单条记忆授予或撤销其他角色的读取权限；相关用户界面尚未完整提供时，系统按上述默认隔离规则处理。

长期记忆可以保存来源会话、消息、说话人、原始引用、时间、版本和关联。纠正事实会保留被替代的历史版本；用户执行删除时，应用会同步清理该记录的证据、关系和索引引用。

## 哪些操作会发送数据

| 操作 | 可能发送的内容 | 接收方 |
|---|---|---|
| 云端模型聊天 | 当前消息、系统提示、所选历史、相关记忆、工具结果和附件 | 用户配置的模型 Endpoint |
| 云端文字转语音 | 待朗读文本、音色、语速和风格参数 | TTS 服务商 |
| 云端语音转文字 | 录音片段和识别参数 | STT 服务商 |
| 在线搜索与网页访问 | 查询词、目标网址和完成任务所需的上下文 | 搜索或网站服务 |
| MCP、Skill、工具包 | 工具参数、选定文件或任务上下文 | 插件配置的本地或远程服务 |
| 主动陪伴通知 | 通知标题、摘要和本地会话标识 | Android 系统通知服务；具体内容由本地生成或所选模型生成 |

请求按用户配置的 Endpoint 发送。若使用中转服务，数据处理边界由该中转服务决定。

## 权限

- **通知**：主动消息、后台任务和语音服务状态。
- **麦克风**：语音输入、STT 和实时通话。
- **文件与媒体**：附件、导入导出、备份和工作区。
- **悬浮窗**：陪伴气泡与快速回复卡片。
- **位置**：仅在用户启用相关能力并调用位置工具时使用。
- **无障碍、Shizuku、Root**：设备协助和自动化。普通文字聊天不依赖这些权限。

用户可以在 Android 系统设置和 Mira 权限页面撤销权限。撤销后，对应能力停止工作，已有本地聊天与记忆不会因此自动删除。

## API Key 与备份

运行时 API 配置保存在应用私有数据中。模型配置导出和完整备份可能包含 API Key、Endpoint、自定义 Header 或其他敏感参数。

当前导出文件不应被视为自动加密备份。Android 应用沙箱只能保护应用私有目录中的数据；文件导出到下载目录、电脑、网盘或聊天工具后，由对应存储和传输方式决定保护强度。

- 使用独立、可撤销、有限额的 Key。
- 不在 Issue、日志、截图、聊天群或公开仓库中粘贴完整 Key。
- 分享备份前先检查内容；不再需要的导出文件应由用户主动删除。
- 怀疑泄露时立即在服务商后台撤销并重新生成 Key。

## 第三方扩展

Mira 兼容 Operit 的 MCP、Skill、脚本和工具包生态。扩展可能访问用户授予的文件、网络、终端或设备工具，也可能连接第三方服务器。

安装前检查作者、源码、更新地址、权限和网络端点。扩展不使用时关闭；来源不明或行为不清楚的扩展不要授予高权限。

## 删除与迁移

用户可以在应用内删除会话、记忆、角色或配置，也可以清除应用数据完成整体删除。导出到应用外部的备份、图片、日志和附件需要在对应存储位置单独删除。

换机时先创建备份并检查内容，再通过用户选择的可信通道传输。导入完成后随机核对会话、记忆和角色，并妥善处理旧备份。

## 联系与变更

隐私问题可通过 [GitHub Issues](https://github.com/kernelx30/Mira/issues) 提交不含敏感数据的说明。安全漏洞或可能暴露用户数据的问题请按 [SECURITY.md](SECURITY.md) 的方式报告。

本说明会随数据结构、服务商接入和权限模型变化而更新。
